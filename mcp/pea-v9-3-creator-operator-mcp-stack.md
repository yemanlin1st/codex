# PEA V9.3 Creator, Marketer, Developer and Business Operator MCP Stack

Date: 2026-06-16
Repository: yemanlin1st/codex
Reference: PEA Account Kernel V9.3

## Purpose

Integrate five additional MCP capability classes inspired by the shared video idea:

- Perplexity MCP for real-time research;
- Playwright MCP for browser automation and QA;
- Firecrawl MCP for structured website crawling;
- Glif MCP for image/video creative generation workflows;
- Chrome MCP for direct browser interaction.

The goal is to make the account more useful for creators, marketers, developers and business owners while keeping strict controls.

## Positioning

These MCP capabilities must not create uncontrolled autonomy. They must operate as a governed AI operating stack.

Formula:

```text
Research + Browser Control + Website Crawl + Creative Generation + Chrome Interaction
+ least privilege + anti-bias + citation + human approval
= useful AI operating system without unsafe overload
```

## Control rules

### Perplexity MCP

- Cite sources.
- Cross-check critical claims.
- Separate facts from assumptions.
- Do not use for legal, medical, financial or regulatory certainty without additional verification.

### Playwright MCP

- Use sandboxed browser context.
- No login with sensitive accounts unless explicitly approved.
- No payments, purchases, destructive actions or production changes.
- Use for testing, screenshots and QA first.

### Firecrawl MCP

- Respect robots, terms, rate limits and copyright.
- Do not scrape personal data.
- Do not bypass paywalls or access controls.
- Use for structured research, audits and public content analysis.

### Glif MCP

- Use for drafts, concepts and creative exploration.
- Do not generate misleading impersonations or deceptive assets.
- Respect brand, copyright and client confidentiality.
- Human approval required before publication.

### Chrome MCP

- Human-supervised mode by default.
- No silent messaging, sending, deleting, purchasing or publishing.
- Read and summarize before acting.
- Use for controlled workflows only.

## Anti-bias and zero-noise integration

Each MCP output must pass:

- anti-bias gate;
- evidence gate;
- privacy/IP gate;
- zero-noise gate;
- human approval gate if action is sensitive.
